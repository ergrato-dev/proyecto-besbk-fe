# RF-014 — Internacionalización (i18n) ES/EN

## Datos generales

| Campo           | Detalle                                              |
| --------------- | ---------------------------------------------------- |
| **ID**          | RF-014                                               |
| **Nombre**      | Internacionalización de la interfaz (español/inglés) |
| **HU asociada** | HU-012                                               |
| **Tipo**        | Frontend + Backend (endpoint de locale)              |

---

## Descripción

La aplicación debe soportar dos idiomas completos (español e inglés) en toda la
interfaz de usuario. El idioma activo se detecta automáticamente, se persiste en
`localStorage` y, para usuarios autenticados, se sincroniza con la columna `locale`
de la base de datos.

---

## Implementación técnica — Frontend

### Stack de i18n

| Paquete                          | Rol                                           |
| -------------------------------- | --------------------------------------------- |
| `i18next`                        | Motor de internacionalización                 |
| `react-i18next`                  | Integración con React (hook `useTranslation`) |
| `i18next-browser-languagedetector` | Detección automática del idioma del navegador |

### Estructura de archivos

```
fe/src/
├── i18n.ts                         # Configuración de i18next
└── locales/
    ├── es/
    │   └── translation.json        # Catálogo en español
    └── en/
        └── translation.json        # Catálogo en inglés
```

### Configuración (`i18n.ts`)

```typescript
// Pseudocódigo de configuración
i18n
  .use(LanguageDetector)      // Detecta idioma del navegador
  .use(initReactI18next)      // Integra con React
  .init({
    resources: { es: { translation: esJSON }, en: { translation: enJSON } },
    fallbackLng: "es",        // Español por defecto
    defaultNS: "translation",
    detection: {
      order: ["localStorage", "navigator", "htmlTag"],
      lookupLocalStorage: "i18nextLng",
      caches: ["localStorage"],
    },
    interpolation: { escapeValue: false },
  });
```

### Uso en componentes

```typescript
// ✅ CORRECTO — hook useTranslation
import { useTranslation } from "react-i18next";

function LoginPage() {
  const { t } = useTranslation();
  return <h1>{t("auth.login.title")}</h1>;
}
```

### Componente LanguageSwitcher

- Botón/toggle visible en `Navbar` en todas las páginas
- Cambia idioma con `i18n.changeLanguage(locale)`
- Muestra el idioma activo (`ES` | `EN`)

## Implementación técnica — Backend

### Migración Flyway `V4__add_locale_to_users.sql`

```sql
-- ¿Qué? Agregar campo locale a la tabla users para persistir la preferencia de idioma.
-- ¿Para qué? Sincronizar la preferencia de idioma del usuario entre dispositivos.
-- ¿Impacto? Sin esta columna, el idioma solo se guarda localmente en el navegador.
ALTER TABLE users
  ADD COLUMN locale VARCHAR(10) NOT NULL DEFAULT 'es';
```

### Actualización de `User.kt`

```kotlin
@Column(name = "locale", nullable = false, length = 10)
var locale: String = "es"
```

### Nuevo endpoint `PATCH /api/v1/users/me/locale`

| Campo          | Detalle                                            |
| -------------- | -------------------------------------------------- |
| **Método**     | `PATCH`                                            |
| **Ruta**       | `/api/v1/users/me/locale`                          |
| **Auth**       | Sí — Bearer token requerido                        |
| **Request**    | `{ "locale": "en" }`                               |
| **Response**   | `200 OK` — `UserResponse` con `locale` actualizado |
| **Validación** | `locale` debe ser `"es"` o `"en"` exactamente      |

### DTO `UpdateLocaleRequest`

```kotlin
data class UpdateLocaleRequest(
    // ¿Qué? @field: dirige la anotación al campo backing del property Kotlin.
    // ¿Para qué? Bean Validation solo funciona si apunta al campo JVM.
    @field:NotBlank(message = "El locale es requerido")
    @field:Pattern(regexp = "^(es|en)$", message = "Locale debe ser 'es' o 'en'")
    val locale: String
)
```

## Reglas de negocio

| ID     | Regla                                                                          |
| ------ | ------------------------------------------------------------------------------ |
| RN-073 | Los únicos locales soportados son `es` (español) y `en` (inglés)               |
| RN-074 | El idioma por defecto es `es` (español)                                        |
| RN-075 | La preferencia se guarda en `localStorage` con clave `i18nextLng`              |
| RN-076 | En la primera visita, se detecta el idioma del navegador (`navigator.language`) |
| RN-077 | Si el idioma del navegador no es soportado, se usa el fallback `es`            |
| RN-078 | Para usuarios autenticados, el cambio de idioma llama `PATCH /users/me/locale` |
| RN-079 | Para usuarios no autenticados, el cambio solo se persiste en `localStorage`    |
| RN-080 | Al hacer login, se sincroniza el `locale` del servidor con el frontend         |
| RN-081 | Todas las páginas (públicas y protegidas) respetan el idioma activo            |
| RN-082 | Los tests deben incluir un mock de i18n en `setup.ts`                          |
